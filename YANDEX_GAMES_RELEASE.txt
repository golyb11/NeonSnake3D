Чеклист публикации «Neon Snake 3D» в Яндекс Играх
================================================

Готовый архив уже лежит в корне как yandex-snake-upload.zip (или пересоздайте: npm run release:yandex).

Подготовка архива (ZIP)
-----------------------
• В архив вкладывайте (без пробелов и кириллицы в именах файлов и папок):

  — index.html
  — style.css
  — game.bundle.js  (единый бандл с Three.js после npm run build)
  — assets/bg-meme-67.png

• НЕ кладите в ZIP папку node_modules (она нужна только на вашем ПК для пересборки бандла).
• Корневой файл запуска: index.html.

Почему была ошибка «ссылка на сервисное хранилище»
---------------------------------------------------
Робот проверки увидел в сборке явные HTTPS-ссылки на внешние хранилища, например:
  — https://sdk.games.s3.yandex.net/sdk.js (бакет S3 «служебного хранилища»);
  — https://unpkg.com/... (чужой CDN для Three.js).
Для игры, залитой через Консоль разработчика, так делать не нужно: SDK подключают относительным путём /sdk.js (платформа сама подставляет скрипт). Движок уже собран локально в game.bundle.js — отдельных CDN в архиве нет.

SDK (актуально)
---------------
В index.html указано:

  <script async src="/sdk.js"></script>

Файла sdk.js в архив добавлять не нужно — на хостинге Яндекс Игр он отдаётся автоматически.
Локально при python -m http.server запрос на /sdk.js даст 404 — это нормально; игра откроется без YaGames (для финальной проверки используйте предпросмотр в Консоли).

Сборка бандла после правок в game.js
------------------------------------
На своей машине:
  npm install
  npm run build

Поднимется актуальный game.bundle.js для новой загрузки в Консоль.

События SDK (в коде)
--------------------
• LoadingAPI.ready() после инициализации.
• GameplayAPI.start/stop при игре, паузе, рекламе и смене вкладки.
• Документация: https://yandex.ru/dev/games/doc/
